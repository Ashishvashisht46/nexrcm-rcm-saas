🏥 NexRCM — Setup Guide

Get your Revenue Cycle Management platform running in 10 minutes. No coding required.

1 Install Docker Desktop 5 min

📥 Download & Install

Docker is like a "container" that runs everything NexRCM needs (database, server, etc.) without you installing each piece separately.

Go to: https://www.docker.com/products/docker-desktop/

Click "Download for Windows" (or Mac), then install it like any normal program — just keep clicking Next.

🔄 Restart Your Computer

After installing Docker Desktop, restart your computer. This is important.

✅ Open Docker Desktop & Wait

After restart, open Docker Desktop. Wait until the bottom-left corner shows a green icon and says "Engine running".

This might take 1-2 minutes the first time. Don't skip this step.

💡 What is Docker? Think of it as a box that contains a mini-computer with everything NexRCM needs pre-installed. You don't need to install databases, servers, or anything else — Docker handles all of it.

2 Start NexRCM 3 min

📂 Open the NexRCM Folder

Go to where you extracted the NexRCM-full-stack-ai.zip file. You should see these files:

📁 nexrcm/ ├── 📄 START-WINDOWS.bat ← Double-click this on Windows! ├── 📄 START-MAC-LINUX.sh ← Run this on Mac/Linux ├── 📄 docker-compose.yml ├── 📄 README.md ├── 📁 backend/ ├── 📁 frontend/ └── 📁 docs/
🖱️ Windows: Double-click START-WINDOWS.bat

That's it. A black command window will open and show the progress. Wait 2-5 minutes the first time (it downloads databases and server components).

If Windows asks "Do you want to allow this?" — click Yes.

🍎 Mac/Linux: Run the script

Open Terminal, navigate to the nexrcm folder, and run:

# Navigate to the folder cd ~/Downloads/nexrcm (or wherever you extracted it) # Make it executable chmod +x START-MAC-LINUX.sh # Run it ./START-MAC-LINUX.sh
🎉 When you see "NexRCM is RUNNING!" — your browser should open automatically. If not, open your browser and go to http://localhost:3000

4 Daily Commands

⏹️ Stop NexRCM

Open a command window (search "cmd" in Windows Start menu) and type:

cd path\to\nexrcm docker-compose down
▶️ Start NexRCM Again (Fast — No Download)
cd path\to\nexrcm docker-compose up -d

After the first setup, starts in ~10 seconds.

📋 See What's Happening (Logs)
docker-compose logs -f

Press Ctrl + C to stop watching logs.

🔄 Reset Everything (Fresh Start)
docker-compose down -v ← deletes all data docker-compose up --build -d ← rebuilds everything fresh

❓ Troubleshooting

🔴 "Docker is not running" error
Open Docker Desktop app and wait for the green "Engine running" status. Then try again.
🔴 Port 3000 or 4000 already in use
Another app is using those ports. Close it, or change the ports in docker-compose.yml (e.g., change "3000:80" to "3001:80").
🔴 "Cannot connect to Docker daemon"
Windows: Right-click the .bat file → "Run as administrator".
Mac/Linux: Run with sudo docker-compose up --build -d
🔴 Browser shows "This site can't be reached"
Wait 1-2 more minutes — the database might still be setting up. Run docker-compose logs -f to check progress. Look for "NexRCM server running on port 4000".
🔴 AI features show "unavailable"
Check that OPENROUTER_API_KEY is set correctly in backend/.env. Verify your key works at openrouter.ai.
🟢 How do I update NexRCM later?
Replace the files, then run: docker-compose down && docker-compose up --build -d
